# Next.js AI协作开发工作流程

> **重要**：本工作流程从 2026-03-26 开始执行，所有Next.js项目的AI协作开发工作必须遵守。

## 📋 概述

本工作流程基于Flutter项目的成功经验，针对Next.js项目特点进行优化，旨在解决AI协作开发中的核心问题：

1. **代码修改看不懂**：确保用户理解每次重要修改的目的和实现
2. **缺乏沟通流程**：建立标准化的沟通和确认机制
3. **记忆不可靠**：通过文档化、清单化、流程化保障规范执行
4. **执行不一致**：建立系统化的执行保障机制

## 🏗️ 工作流程图

```
用户请求 → AI分析请求 → 判断重要性 →
    ↓ (重要修改)                    ↓ (常规修改)
启动沟通流程                    直接实施
    ↓                            ↓
问题陈述阶段 → 用户确认        实施完成
    ↓                            ↓
方案设计阶段 → 用户确认        结果报告
    ↓
风险评估阶段 → 用户确认
    ↓
实施授权阶段 → 用户明确同意
    ↓
开始实施 → 进度报告 → 完成总结
```

## 🔍 各阶段详细要求

### 阶段1：请求分析

#### 1.1 分析内容

- 分析用户请求的复杂度和重要性
- 判断是否需要走完整沟通流程
- 识别Next.js特定技术挑战（SSR/CSR、环境变量、部署配置等）

#### 1.2 判断标准

**需要完整沟通的重要修改包括**：

- ✅ 架构级设计变更（App Router vs Pages Router、渲染模式变更）
- ✅ 核心业务逻辑修改（支付、订单、用户数据）
- ✅ 重大性能优化（图片优化、代码分割、缓存策略）
- ✅ 安全相关修改（CSP、认证授权、环境变量管理）
- ✅ 第三方服务集成（Sentry、Lighthouse、Cloudflare Workers）
- ✅ 代码结构重大重构（组件拆分、状态管理变更）

**可以直接实施的常规修改**：

- 🔧 Bug修复（明显的UI错误、类型错误）
- 📝 文档更新（注释和文档完善）
- 🎨 UI微调（不影响功能的样式调整）
- 🔄 依赖更新（版本升级和兼容性调整）

### 阶段2：沟通流程（重要修改）

#### 2.1 问题陈述阶段

**目标**：确保用户理解修改的背景和必要性

**必须包含的内容**：

- **问题描述**：当前存在什么问题？
- **影响范围**：这个问题影响了哪些页面或功能？
- **Next.js特定考量**：是否涉及SSR/CSR、环境变量、部署配置？
- **修改目标**：希望通过修改达到什么效果？

**沟通模板**：
必须使用 `docs/templates/IMPORTANT_CODE_CHANGE_TEMPLATE.md` 模板

**用户确认要求**：

- 等待用户对问题陈述的确认
- 根据用户反馈调整问题描述
- 获得继续下一步的授权

#### 2.2 方案设计阶段

**目标**：确保用户理解技术方案和实现思路

**必须包含的内容**：

- **技术方案**：计划如何解决这个问题？
- **Next.js实现细节**：使用App Router还是Pages Router？SSR还是CSR？
- **环境变量处理**：是否需要新增或修改环境变量？
- **部署影响**：对Cloudflare Workers部署有什么影响？

**技术方案要求**：

- 必须考虑现有架构的兼容性
- 必须评估技术实现的可行性
- 必须提供备选方案供用户选择

**用户确认要求**：

- 等待用户对技术方案的确认
- 根据用户反馈调整方案
- 获得继续下一步的授权

#### 2.3 风险评估阶段

**目标**：确保用户理解修改的风险和应对措施

**必须包含的内容**：

- **技术风险**：可能遇到的技术挑战？
- **Next.js特定风险**：SSR水合错误、环境变量泄露、构建失败
- **兼容性风险**：对其他模块的影响？
- **部署风险**：对Cloudflare Workers部署的影响？
- **回滚方案**：如果出现问题如何回滚？
- **监控方案**：如何监控修改效果？

**风险评估要求**：

- 必须识别所有潜在风险
- 必须制定相应的应对措施
- 必须提供风险等级评估

**用户确认要求**：

- 等待用户对风险评估的确认
- 根据用户反馈调整风险应对措施
- 获得继续下一步的授权

#### 2.4 实施授权阶段

**目标**：获得用户明确的实施授权

**必须包含的内容**：

- **最终方案总结**：汇总所有阶段的确认结果
- **实施时间安排**：明确的实施时间表
- **沟通计划**：实施过程中的沟通安排
- **成功标准**：如何判断修改成功

**授权要求**：

- 必须获得用户明确的"同意实施"确认
- 必须记录授权时间和条件
- 必须确认沟通渠道和频率

### 阶段3：实施阶段

#### 3.1 实施前准备

**必须完成的检查**：

- [ ] 是否已完成所有沟通阶段的用户确认？
- [ ] 是否已查阅 `docs/nextjs/COMMANDS_CHEATSHEET.md`？
- [ ] 是否已准备好必要的开发环境？
- [ ] 是否已制定详细的实施步骤？
- [ ] 是否已检查环境变量配置？
- [ ] 是否已考虑部署影响？

#### 3.2 实施过程

**实施要求**：

- 按照确认的方案实施
- 定期报告实施进度（至少每30分钟报告一次）
- 遇到问题立即暂停并沟通
- 记录所有实施的命令和操作

**命令执行规范**：

- 所有命令必须在沟通中明确写出
- 必须记录命令的执行目的和预期结果
- 必须验证命令的实际执行结果
- 如果命令失败，必须分析原因并记录

#### 3.3 进度报告

**报告内容**：

- 当前实施进度
- 已完成的工作
- 遇到的问题和解决方案
- 下一步计划
- 预计完成时间

**报告频率**：

- 重要修改：至少每30分钟报告一次
- 常规修改：实施完成后报告

### 阶段4：完成阶段

#### 4.1 完成总结

**必须包含的内容**：

- **实施结果总结**：实际完成的工作
- **效果验证**：如何验证修改效果
- **问题记录**：实施过程中遇到的问题和解决方案
- **经验总结**：从这次实施中学到的经验

#### 4.2 文档更新

**必须更新的文档**：

- [ ] 更新 `.github/copilot-instructions.md` 中的任务状态
- [ ] 更新相关技术文档
- [ ] 记录实施经验和教训
- [ ] 更新命令速查表（如果有新的命令）

#### 4.3 用户验收

**验收要求**：

- 提供完整的实施报告
- 演示修改效果
- 回答用户的问题
- 获得用户的最终验收确认

### 文档引用机制

#### 必须引用的文档：

2. **工作流程文档**：`docs/nextjs/AI_COLLABORATION_WORKFLOW.md`（重要修改前必读）

#### 文档更新要求：

- 每次重要修改后必须更新相关文档
- 文档冲突时以代码为准，然后更新文档
- 文档必须保持与代码同步

### 违规处理机制

#### 违规类型：

1. **一级违规**：未完成沟通流程的重要修改
2. **二级违规**：未查阅必要文档的命令执行
3. **三级违规**：未记录实施过程和结果

#### 处理措施：

1. **立即暂停**：发现违规立即暂停当前工作
2. **重新沟通**：按照规范重新开始沟通流程
3. **记录违规**：在文档中记录违规情况和处理措施
4. **流程调整**：多次违规将调整工作流程

## 🚀 实施指南

### Next.js特定注意事项

#### 环境变量管理：

- 开发环境：`.env.development`
- 生产环境：`.env.production`
- 测试环境：`.env.test`
- 必须使用 `NEXT_PUBLIC_` 前缀公开客户端变量

#### 部署配置：

- Cloudflare Workers部署：`opennextjs-cloudflare`
- 环境变量注入：通过GitHub Secrets
- 健康检查：`CF_ADMIN_HEALTHCHECK_URL`

#### 性能优化：

- 图片优化：使用Next.js Image组件
- 代码分割：动态导入
- 缓存策略：ISR、SSR缓存

### 常见问题：

1. **Q：如何判断修改是否重要？**
   A：参考"阶段1：请求分析"中的判断标准

2. **Q：沟通需要多长时间？**
   A：重要修改通常需要30-60分钟的沟通时间

3. **Q：如果用户不回复怎么办？**
   A：等待用户回复，不要继续实施。可以适当提醒用户。

### 效率优化建议

#### 沟通效率：

1. **提前准备**：在沟通前准备好所有必要信息
2. **结构化沟通**：使用标准模板进行结构化沟通
3. **明确问题**：一次沟通解决一个问题
4. **及时确认**：每个阶段完成后及时获得用户确认

#### 实施效率：

1. **分步实施**：将大修改分解为小步骤
2. **定期报告**：定期报告进度，避免长时间无沟通
3. **问题预警**：提前预警可能的问题
4. **经验复用**：复用之前的成功经验
